# 一、树的重心（Tree Centroid）

## 1.1 定义

对于一棵 n 个节点的无根树，找到一个节点 c，使得**删去 c 及其相连边后**，剩余各连通块（子树）中**最大子树的节点数最小**。这个节点 c 就是树的重心。

### 关键性质

1. **树的重心存在且不超过 2 个**。若有 2 个，它们必相邻。
2. **重心的判定**：以任意节点为根，设 `sz[u]` 为 u 的子树大小，则 u 是重心 ⟺ 对 u 的所有子节点 v，都有 `sz[v] ≤ n/2`，且 `n - sz[u] ≤ n/2`（父方向的连通块也不超过一半）。
3. **核心性质**：删去重心后，最大连通块大小 ≤ ⌊n/2⌋。这意味着每次"从重心切一刀"，问题规模至少减半，总递归深度为 O(log n)。

### 直觉理解

> 重心就是树的"平衡点"。想象把树挂在重心上，各方向的"重量"（节点数）最均衡。
> 类比：三角形的重心是三条中线的交点，树的重心是让最大子树最小的节点。

## 1.2 求法

一次 DFS 即可。以任意节点（如 1）为根，计算每个节点的子树大小 `sz[u]`，然后对每个节点 u 检查：

```
max_part[u] = max( max{ sz[v] for v in children(u) },  n - sz[u] )
```

重心就是 `max_part` 最小的那个（些）节点。

### 代码模板

```cpp
const int N = 6e4 + 5;
vector<int> adj[N];
int sz[N];       // 子树大小
int max_part[N]; // 删去该节点后最大连通块大小
int centroid;    // 重心编号
int best;        // 最小的 max_part 值
int n;

void dfs_centroid(int u, int fa) {
    sz[u] = 1;
    max_part[u] = 0;
    for (int v : adj[u]) {
        if (v == fa) continue;
        dfs_centroid(v, u);
        sz[u] += sz[v];
        max_part[u] = max(max_part[u], sz[v]);
    }
    max_part[u] = max(max_part[u], n - sz[u]); // 父方向的连通块
    if (max_part[u] < best) {
        best = max_part[u];
        centroid = u;
    }
}

// 调用：
// best = n;
// dfs_centroid(1, 0);
// centroid 即为重心
```

**时间复杂度**：O(n)，一次 DFS 遍历所有节点。

## 1.3 为什么重心如此重要？

重心保证了"每次切一刀，问题规模至少减半"。这是点分治复杂度 O(n log n) 的根基。没有这个性质，递归可能退化成链式的 O(n²)。

# 二、点分治（Centroid Decomposition）

## 2.1 核心思想

> **点分治 = 在重心处统计经过它的所有路径 + 递归处理剩余子树**

面对"统计树上所有路径"类问题，暴力枚举所有路径是 O(n²)。点分治的思路：

1. 找到当前树的重心 c
2. **治**：统计所有**经过重心 c** 的路径的贡献
3. **分**：删去重心 c，对剩下的每个连通块（子树）递归执行步骤 1

### 为什么这样做高效？

- 每条路径恰好在某一层被重心"截断"（恰好经过该层的重心），因此每条路径只被统计一次
- 每次递归问题规模至少减半，递归深度 O(log n)
- 每层处理的总节点数为 O(n)，所以总复杂度 O(n log n)（不含统计内部的排序等操作）

## 2.2 算法框架

```
function solve_tree(T):
    if T 为空: return
    c = find_centroid(T)           // 找重心
    count_paths_through(c)         // 统计经过 c 的路径贡献
    删去 c
    for each 连通块 S (c 的每个邻居子树):
        solve_tree(S)              // 递归处理
```

## 2.3 详细实现

### 框架代码

```cpp
const int N = 6e4 + 5;
vector<int> adj[N];
bool removed[N];  // 标记已删除的重心
int sz[N], max_part[N];
int n;

// ====== 第一步：求子树大小（用于找重心）======
void calc_size(int u, int fa) {
    sz[u] = 1;
    max_part[u] = 0;
    for (int v : adj[u]) {
        if (v == fa || removed[v]) continue;
        calc_size(v, u);
        sz[u] += sz[v];
        max_part[u] = max(max_part[u], sz[v]);
    }
}

// ====== 第二步：找重心 ======
int find_centroid(int u, int fa, int total) {
    max_part[u] = max(max_part[u], total - sz[u]);
    int cen = u;
    for (int v : adj[u]) {
        if (v == fa || removed[v]) continue;
        int sub = find_centroid(v, u, total);
        if (max_part[sub] < max_part[cen]) {
            cen = sub;
        }
    }
    return cen;
}

// ====== 第三步：统计经过重心的路径 ======
void count_paths(int u, int fa, ...) {
    // 收集从重心出发到 u 的路径信息（max、min 等）
    for (int v : adj[u]) {
        if (v == fa || removed[v]) continue;
        count_paths(v, u, ...);
    }
}

void collect(int centroid) {
    // 以重心为根，遍历每个子树
    for (int v : adj[centroid]) {
        if (removed[v]) continue;
        count_paths(v, centroid, ...);
        // 将本子树收集的路径信息加入"已有集合"，统计跨子树路径对
    }
}

// ====== 主函数：点分治 ======
void divide(int u) {
    calc_size(u, 0);
    int c = find_centroid(u, 0, sz[u]);
    removed[c] = true;

    // 统计经过重心 c 的路径
    collect(c);

    // 递归处理各子树
    for (int v : adj[c]) {
        if (!removed[v]) {
            divide(v);
        }
    }
}

// 调用：divide(1);
```

### ⚠️ 注意事项

1. **`removed[]` 数组**：标记已作为重心被删除的节点，遍历时跳过
2. **找重心时 `total` 参数**：是当前连通块的总大小，不是整棵树的 n
3. **统计时排除同子树路径**：路径必须经过重心，因此两端在不同子树中。处理时先统计再加入，避免同一子树内的路径被错误统计

## 2.4 复杂度分析

- **递归深度**：O(log n)，因为每次问题规模至少减半
- **每层处理**：遍历当前连通块所有节点，O(size)
- **总复杂度**：O(n log n)（不含排序等附加操作）

# 三、路径极值追踪

## 3.1 问题描述

在点分治的"统计经过重心的路径"步骤中，我们需要对每条从重心 c 出发到节点 t 的路径，维护：

- `max_val[t]`：路径 c → t 上的最大点权
- `min_val[t]`：路径 c → t 上的最小点权
- `depth[t]`：路径长度（可选）

### DFS 过程中维护

```cpp
struct PathInfo {
    int max_val; // 路径上的最大值
    int min_val; // 路径上的最小值
};

vector<PathInfo> paths; // 收集到的所有路径信息

void dfs_collect(int u, int fa, int cur_max, int cur_min) {
    cur_max = max(cur_max, a[u]);
    cur_min = min(cur_min, a[u]);
    paths.push_back({cur_max, cur_min});

    for (int v : adj[u]) {
        if (v == fa || removed[v]) continue;
        dfs_collect(v, u, cur_max, cur_min);
    }
}
```

重心 c 本身的路径信息为 `{a[c], a[c]}`，需要单独加入。

## 3.2 合并两条路径

当路径 p1（s → c）和路径 p2（c → t）合并为完整路径 s → t 时：

```
combined_max = max(p1.max_val, p2.max_val)
combined_min = min(p1.min_val, p2.min_val)
Val(s,t)     = combined_max - combined_min
```

**注意**：不要忘记把重心 c 自身的权值 `a[c]` 也包含在内！

# 四、二维约束计数技巧

## 4.1 问题形式

经过重心 c 的路径，由两段子路径拼接而成。对于路径对 (p1, p2)，我们有：

```
combined_max = max(p1.max, p2.max)
combined_min = min(p1.min, p2.min)
range = combined_max - combined_min
```

我们需要统计满足 `|a_s - a_t| ≤ l` 的路径对的 `range` 之和。

## 4.2 排序 + 扫描统计

### 思路

将所有路径信息按 `max_val` 排序。然后分两种情况讨论 `combined_max` 来自哪一侧。

**情况 A**：`combined_max = p2.max`（即 p2.max ≥ p1.max）

此时 `range = p2.max - combined_min`。对固定的 p2，需要找满足条件的 p1（max ≤ p2.max）使得：

```
combined_min = min(p1.min, p2.min)
range = p2.max - combined_min
```

再细分：
- 若 `p1.min ≥ p2.min`：`combined_min = p2.min`，`range = p2.max - p2.min`（与 p1 无关）
- 若 `p1.min < p2.min`：`combined_min = p1.min`，`range = p2.max - p1.min`

**利用排序 + 树状数组/双指针**：按 max 排序后，依次处理每条路径。用数据结构维护已处理路径的 min 值分布，支持：
- 统计 min ≥ threshold 的路径数量
- 统计 min < threshold 的路径的 min 值之和

### 代码思路（使用树状数组）

```cpp
// 将所有路径按 max_val 排序
// 依次处理每条路径 p[i]（max 从小到大）
// 维护两个树状数组：
//   bit_cnt: 记录 min 值的计数
//   bit_sum: 记录 min 值的和

// 对路径 p[i]：
//   在 bit_cnt / bit_sum 中查询已有路径中 min 的分布
//   计算贡献
//   将 p[i] 的 min 值插入 bit_cnt / bit_sum
```

### 离散化

由于 min 值的范围可能很大，需要先**离散化**（将所有可能的 min 值映射到 1~m 的紧凑下标）。

```cpp
// 离散化模板
vector<int> vals; // 收集所有可能的值
sort(vals.begin(), vals.end());
vals.erase(unique(vals.begin(), vals.end()), vals.end());
int get_rank(int x) {
    return lower_bound(vals.begin(), vals.end(), x) - vals.begin() + 1;
}
```

## 4.3 同子树排除

在统计时，必须确保路径对的两端在重心的不同子树中。

处理顺序：
1. 维护一个"全局"的数据结构 `global_bit`
2. 对重心的每个子树 S：
   a. 收集 S 中所有路径到 `local_paths`
   b. 对 `local_paths` 中的每条路径，与 `global_bit` 中的路径配对统计
   c. 将 `local_paths` 中的路径插入 `global_bit`
3. 最后别忘了重心自身的路径（单独的节点）

这样保证每对路径来自不同子树。

## 4.4 完整统计流程（伪代码）

```
function collect(centroid):
    global_bit_cnt = empty Fenwick tree
    global_bit_sum = empty Fenwick tree
    
    // 插入重心自身路径 {a[c], a[c]}
    insert(global_bit_cnt, global_bit_sum, min=a[c])
    
    for each child subtree S of centroid:
        local_paths = []
        dfs_collect(S, centroid, a[c], a[c], local_paths)
        
        // 统计 local_paths 中的路径与 global_bit 中的路径的贡献
        for each path p in local_paths:
            count_and_sum(p, global_bit_cnt, global_bit_sum)
        
        // 将 local_paths 插入 global_bit
        for each path p in local_paths:
            insert(global_bit_cnt, global_bit_sum, p.min)
```

# 六、点分治的本质理解

## 6.1 为什么叫"点分治"？

- **点**：每次以一个**点**（重心）为切入点
- **分**：删去重心后，问题**分**裂为若干独立子问题
- **治**：在重心处**治**理（统计）跨越不同子树的路径

对比：
| 方法 | 切割方式 | 统计内容 |
|------|---------|---------|
| 归并分治 | 数组中点 | 跨越中点的逆序对等 |
| 点分治 | 树的重心 | 经过重心的路径 |

## 6.2 点分治 vs 树上 DP

| 维度 | 点分治 | 树上 DP |
|------|--------|---------|
| 视角 | 全局枚举路径 | 自底向上合并子树信息 |
| 适用场景 | 统计所有路径 | 统计与根相关的路径 |
| 复杂度 | O(n log n) | O(n) |
| 局限 | 需要高效统计手段 | 状态设计困难 |

## 6.3 常见点分治题型

- 路径长度 ≤ k 的路径数量
- 路径权值和 = k 的路径数量
- 路径上第 k 大/小
- 路径极差（max - min）相关
- 路径上的异或值相关
