### 累加和线段树构造
#### 初始化
1. 下标从1开始(也可以从0开始)
2. 线段树需要在初始化时确定数据规模$[1,n]$,确定后不能更改
3. 任何一个存在2个数及以上的范围$[l,r](l<r)$，拆分为左右两个范围$[l,mid]$,$[mid+1,r]$（其中$mid=(l+r)/2$,向下取整）
4. 逻辑结构用满二叉树存放数据，对于节点i，他的左范围的结果存放在$2*i$节点上，右范围的结果存在$2*i+1$节点上。
5. 由于满二叉树的良好性质，储存结构可以用数组存储，每个范围的信息填写在独立的、连续数组sum(用于储存范围累加和，查询信息的数组)中，最大范围$[1,n]$,信息填写在$sum[1]$中，用归并分治的思想，先确定每个节点填要什么，分到不能再分的时候开始从小范围到大范围逐步填写左范围与右范围的累加和
6. 范围与i值一一对应，受公式严格限制，由递归参数维护，不需要单独记录对应关系，可能存在一定的空间浪费
7. 对于范围为$[1,n]$的数据，开$4*n$的sum数组足够，证明如下：对于数据n，有它形成的满二叉树的高度向上取整为$h=log2*n+1$(底数都为2),那么节点数总和为$2^h-1$,带入可得$4*n-1$，取$4*n$即可
8. 整个结构是一个区间不断二分的树形结构，大区间在上，不断二分的小区间顺着树深度排下去
```cpp
void build(int l,int r,int i){
	if(l==r) sum[i]=arr[l];  // 分无可分的时候更新
	else{
		int mid=(l+r)>>1;
		build(l,mid,i<<1);
		build(mid+1,r,i<<1|1); // 分治
		up(i);        // 父区间=左子区间+右子区间
	}
	buffer[i]=0;
}
```
#### 区间修改
- 添加lazy缓存数组（用于短暂存放修改记录）
- 区间修改的操作与时机：
	1. 上传操作 往往在分治之后回调以前，当左右子树都更新好后信息汇总，左右更新好的区间合并，整个操作是从线段树由底向上更新
	2. 直接更新 当修改区间包含当前区间的时候直接更新当前子区间，至于父区间的修改留到上传操作更新，整个操作只在线段树当前层更新
	3. 下载操作 当这个子区间有必要更新的时候，根据缓存数组的数往下一层的子区间更新，整个操作由上而下更新
 ```cpp
 
 // 上传操作  左子区间与右子区间合并
 void up(int i){
	sum[i]=sum[i<<1]+sum[i<<1|1];
}
 ```

 ```cpp
  // 直接修改，修改的是一个子区间
void apply(int i,ll v,int n){
	// 对于累加和数组sum，数据更新应该是 单个数字更新大小v*区间大小n
	sum[i]+=v*n;
	// 对于缓存数组lazy，只需要标记，表示该的子区间区间存在一个修改缓存，非必要不完全更新
	lazy[i]+=v;
}
 ```

```cpp
// 下载操作，修改的是子区间
void down(int i,int ln,int rn){
	if(lazy[i]!=0){
		apply(i<<1,lazy[i],ln);
		apply(i<<1|1,lazy[i],rn);
		lazy[i]=0;  // 清除这个缓存
	}
}
```
 - 懒更新机制(缓存机制) 只修改当前区间的内容，然后通过上传更新父区间的内容，至于子区间的内容，先挂缓存（存到lazy数组中），如无必要不更新，即使更新也只更新一层（或只更新到需要的层数）；当且仅当两种情况需要更新：
	 1. 当修改区间与当前区间的子区间存在重叠关系，需要分治到更小的区间时；
	 2. 当查询区间与当前区间的子区间存在重叠关系，需要分治到更小的区间时。
```cpp
void add(int jl,int jr,ll v,int l,int r,int i){
	if(jl<=l&&r<=jr){  
		// 只有当前区间被修改区间完全包含才立即更新，并挂上缓存
		lazy(i,v,r-l+1); 
	}
	else{
		// 如果修改区间没有完全包含当前区间，清空当前层的缓存，然后分成左右区间，检查左右区间是否存在被修改空间包含的空间             
		int mid=((l+r)>>1);
		down(i,mid-l+1,r-mid);
		if(jl<=mid){
			add(jl,jr,v,l,mid,i<<1);
		}
		if(jr>mid){
			add(jl,jr,v,mid+1,r,i<<1|1);
		}
		up(i);
	}
}
```
#### 区间查询
只围绕查询区间的左右边界往树下方查询，并由底向顶更新答案，O($logn$)级别，当查询操作深入到有缓存未更新的子区间的时候，更新缓存
```cpp
ll query(int jl,int jr,int l,int r,int i){
	if(jl<=l&&r<=jr){
		return sum[i];
	}
	int mid=((l+r)>>1);
	ll ans=0;
	down(i,mid-l+1,r-mid); // 更新缓存
	if(jl<=mid){
		ans+=query(jl,jr,l,mid,i<<1);
	}
	if(jr>mid){
		ans+=query(jl,jr,mid+1,r,i<<1|1);
	}
	return ans;
}
```
