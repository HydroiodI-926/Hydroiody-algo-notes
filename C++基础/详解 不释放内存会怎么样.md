# 不释放内存的严重后果

## 1. 内存泄漏 (Memory Leak)

### 什么是内存泄漏？
程序申请了内存但忘记释放，导致这部分内存无法被再次使用。

### 简单示例
```cpp
void memoryLeak() {
    int* arr = new int[1000];  // 申请内存
    // 使用数组...
    // 忘记 delete[] arr;  ← 内存泄漏！
}

int main() {
    for(int i = 0; i < 10000; i++) {
        memoryLeak();  // 每次调用都泄漏 1000*4 bytes
    }
    // 程序运行后，大量内存无法回收
    return 0;
}
```

## 2. 内存泄漏的后果

### 短期后果
```cpp
// 小规模泄漏可能不明显
void smallLeak() {
    int* data = new int[10];  // 泄漏40字节
    // 无delete
}

// 但循环调用会产生累积效应
int main() {
    while(true) {
        smallLeak();  // 持续泄漏，内存不断增长
    }
}
```

### 长期后果
1. **程序内存占用持续增长**
2. **系统变慢**：可用内存减少
3. **程序崩溃**：内存耗尽时
4. **影响其他程序**：系统资源紧张

## 3. 实际演示

### 内存增长可视化
```cpp
#include <iostream>
using namespace std;

void leakingFunction() {
    // 每次调用泄漏 1MB
    char* memory = new char[1024 * 1024];  // 1MB
    // 故意不释放
}

int main() {
    cout << "开始内存泄漏演示..." << endl;
    
    for(int i = 1; i <= 10; i++) {
        leakingFunction();
        cout << "已泄漏: " << i << " MB" << endl;
        
        // 在Windows中可以用系统命令查看内存变化
        // system("pause");  // 暂停观察
    }
    
    cout << "程序结束，但泄漏的内存无法回收!" << endl;
    return 0;
}
```

## 4. 不同类型程序的影响

### 短期运行的程序（算法竞赛）
```cpp
// 在算法竞赛中，偶尔的小泄漏可能不会被察觉
// 因为程序运行时间短，结束后操作系统会回收所有内存

void solve() {
    int* temp = new int[1000000];  // 临时大数组
    // 计算...
    // 忘记 delete[] temp;  ← 在竞赛中可能不会立即出问题
}

// 但要注意：如果题目数据量大或循环次数多，仍会导致内存超限(MLE)
```

### 长期运行的程序（服务器、应用程序）
```cpp
// 服务器程序必须严格管理内存
class Server {
public:
    void handleRequest() {
        // 每次请求都分配内存
        RequestData* data = new RequestData();
        processRequest(data);
        // 如果忘记 delete data;
        // 服务器运行几天后就会因内存耗尽崩溃
    }
};
```

## 5. 内存泄漏的检测

### 简单检测方法
```cpp
#include <iostream>
using namespace std;

// 全局计数器（简单检测）
int allocationCount = 0;

void* operator new(size_t size) {
    allocationCount++;
    cout << "分配 " << size << " 字节，总分配次数: " << allocationCount << endl;
    return malloc(size);
}

void operator delete(void* memory) noexcept {
    allocationCount--;
    cout << "释放内存，剩余分配: " << allocationCount << endl;
    free(memory);
}

int main() {
    int* arr1 = new int[100];    // 计数+1
    int* arr2 = new int[200];    // 计数+1
    delete[] arr1;               // 计数-1
    // 忘记 delete[] arr2;       // 内存泄漏！计数保持+1
    
    cout << "程序结束，未释放的分配: " << allocationCount << endl;
    return 0;
}
```

## 6. 避免内存泄漏的最佳实践

### 方法1：使用智能指针（推荐）
```cpp
#include <memory>
using namespace std;

void safeFunction() {
    // unique_ptr 自动管理内存
    unique_ptr<int[]> arr(new int[1000]);
    // 不需要手动delete，离开作用域时自动释放
}

void safeFunction2() {
    // 使用make_unique (C++14)
    auto arr = make_unique<int[]>(1000);
    // 更安全，避免显式new
}
```

### 方法2：RAII模式
```cpp
class ManagedArray {
private:
    int* data;
    size_t size;
public:
    ManagedArray(size_t n) : size(n), data(new int[n]) {}
    ~ManagedArray() { delete[] data; }  // 析构时自动释放
    
    // 禁用拷贝（或实现深拷贝）
    ManagedArray(const ManagedArray&) = delete;
    ManagedArray& operator=(const ManagedArray&) = delete;
};

void safeFunction() {
    ManagedArray arr(1000);  // 构造时分配
    // 使用arr...
} // 离开作用域时自动调用析构函数释放内存
```

### 方法3：严格遵守 new/delete 配对
```cpp
void carefulFunction() {
    int* arr = nullptr;
    try {
        arr = new int[1000];
        // 使用数组...
        
        delete[] arr;  // 明确释放
        arr = nullptr; // 避免悬空指针
    } catch(...) {
        delete[] arr;  // 异常时也要释放
        throw;
    }
}
```

## 7. 总结

**不释放内存的后果：**
- ✅ **短期小程序**：可能无立即影响（操作系统会回收）
- ❌ **长期运行程序**：内存耗尽，程序崩溃
- ❌ **大规模程序**：系统性能下降
- ❌ **专业开发**：被视为严重bug

**黄金法则：** 每个 `new` 都要有对应的 `delete`，每个 `new[]` 都要有对应的 `delete[]`！

在现代C++中，**尽量使用智能指针和容器**，避免手动内存管理。
