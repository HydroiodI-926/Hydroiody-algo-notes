# 数组遍历的三种方式详解

## 方式1：使用下标访问（最常用）

```cpp
int arr[5] = {10, 20, 30, 40, 50};

for(int i = 0; i < 5; i++) {
    cout << arr[i] << " ";
}
// 输出：10 20 30 40 50
```

**工作原理：**
- `arr[i]` 是语法糖，编译器会转换为：`*(arr + i)`
- `i` 是偏移量，表示距离数组开头的元素个数
- 每次循环访问 `arr + i` 位置的值

**内存访问过程：**
```
arr[0] → 地址: arr + 0 → 值: 10
arr[1] → 地址: arr + 1 → 值: 20  
arr[2] → 地址: arr + 2 → 值: 30
arr[3] → 地址: arr + 3 → 值: 40
arr[4] → 地址: arr + 4 → 值: 50
```

**优点：** 直观易懂，最常用

## 方式2：使用指针算术

```cpp
int arr[5] = {10, 20, 30, 40, 50};

for(int i = 0; i < 5; i++) {
    cout << *(arr + i) << " ";
}
// 输出：10 20 30 40 50
```

**详细分解：**
- `arr`：数组名，是指向第一个元素的指针（地址值）
- `arr + i`：指针算术，移动到第 i 个元素的地址
- `*(arr + i)`：解引用，获取该地址的值

**指针算术的底层：**
```cpp
// arr + i 的实际计算：
地址 = arr的起始地址 + i * sizeof(int)

// 假设arr起始地址是1000，int占4字节：
arr + 0 = 1000 + 0*4 = 1000 → *(1000) = 10
arr + 1 = 1000 + 1*4 = 1004 → *(1004) = 20  
arr + 2 = 1000 + 2*4 = 1008 → *(1008) = 30
// 以此类推...
```

**编译器视角：**
```cpp
// 这两种写法完全等价：
arr[i]    ⇔    *(arr + i)
```

## 方式3：使用指针变量遍历

```cpp
int arr[5] = {10, 20, 30, 40, 50};

int* ptr = arr;  // ptr指向数组开头
for(int i = 0; i < 5; i++) {
    cout << *ptr << " ";
    ptr++;  // 移动到下一个元素
}
// 输出：10 20 30 40 50
```

**逐步分析：**

**初始状态：**
```cpp
int* ptr = arr;  // ptr指向arr[0]
// ptr存储的地址 = arr的地址
// *ptr = 10
```

**第一次循环：**
```cpp
cout << *ptr << " ";  // 输出: 10
ptr++;                // ptr移动到下一个int的位置
// 现在ptr指向arr[1], *ptr = 20
```

**第二次循环：**
```cpp
cout << *ptr << " ";  // 输出: 20  
ptr++;                // ptr指向arr[2], *ptr = 30
```

**继续直到结束...**

**内存变化示意图：**
```
初始: ptr → [10][20][30][40][50]
循环1: 输出10, ptr++ → [10][20][30][40][50]
                            ↑
                            ptr
循环2: 输出20, ptr++ → [10][20][30][40][50]
                                 ↑
                                 ptr
...
```

## 三种方式的对比

### 语法对比
```cpp
// 方式1：下标
arr[i]

// 方式2：指针算术  
*(arr + i)

// 方式3：指针遍历
*ptr 然后 ptr++
```

### 性能考虑
- 现代编译器会对三种方式生成相似的优化代码
- 方式1最易读，方式3在某些情况下可能稍快（减少地址计算）

### 可读性
- **方式1**：最高，最符合直觉
- **方式2**：中等，显示展示了指针关系  
- **方式3**：较低，需要理解指针移动

## 变体写法

### 方式3的另一种写法（使用指针比较）
```cpp
int arr[5] = {10, 20, 30, 40, 50};
int* ptr = arr;
int* end = arr + 5;  // 结束位置

while(ptr < end) {
    cout << *ptr << " ";
    ptr++;
}
// 输出：10 20 30 40 50
```

### 结合使用
```cpp
int arr[5] = {10, 20, 30, 40, 50};

// 混合方式：用指针但使用下标语法
int* ptr = arr;
for(int i = 0; i < 5; i++) {
    cout << ptr[i] << " ";  // ptr[i] 等价于 *(ptr + i)
}
```

## 重要概念：数组名与指针的关系

```cpp
int arr[5] = {10, 20, 30, 40, 50};

cout << arr << endl;        // 数组首地址
cout << &arr[0] << endl;    // 第一个元素的地址（与arr相同）
cout << *arr << endl;       // 第一个元素的值：10

// 数组名在大多数情况下会退化为指向首元素的指针
// 但有一个例外：sizeof(arr) 返回整个数组的大小
```

## 总结

- **方式1**：日常编程首选，清晰易懂
- **方式2**：理解数组与指针关系的桥梁
- **方式3**：底层操作，在特定优化场景有用

**关键理解：** 在C++中，数组访问本质上都是通过指针算术实现的，只是语法糖让我们可以用更直观的方式编写代码。
