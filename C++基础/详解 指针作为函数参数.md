# 函数参数传递方式详解

## 三种参数传递方式的本质区别

### 1. 传值 (Pass by Value)
```cpp
void incrementByValue(int x) {
    x++;  // 修改的是副本，不影响原变量
}

int main() {
    int num = 10;
    incrementByValue(num);
    cout << num << endl;  // 输出：10
}
```

**内存过程：**
1. 调用 `incrementByValue(num)` 时，创建 `num` 的**副本**
2. 函数内部操作的是这个**副本**
3. 函数返回时，副本被销毁
4. 原始 `num` **完全不受影响**

**比喻：** 就像复印一份文件，在复印件上修改，原件不变。

### 2. 传指针 (Pass by Pointer)
```cpp
void incrementByPointer(int* x) {
    (*x)++;  // 通过指针找到原变量并修改
}

int main() {
    int num = 10;
    incrementByPointer(&num);  // 传递num的地址
    cout << num << endl;       // 输出：11
}
```

**内存过程：**
1. `&num` 获取 `num` 的内存地址
2. 将这个**地址值**传递给函数
3. 函数通过 `*x` 找到原始变量并修改
4. 原始 `num` **被直接修改**

**比喻：** 就像给别人你家的钥匙，他们可以直接进你家修改东西。

### 3. 传引用 (Pass by Reference)
```cpp
void incrementByReference(int& x) {
    x++;  // 直接操作原变量
}

int main() {
    int num = 10;
    incrementByReference(num);  // 直接传递变量
    cout << num << endl;        // 输出：12
}
```

**内存过程：**
1. `x` 是 `num` 的**别名**（引用）
2. 对 `x` 的任何操作都直接作用在 `num` 上
3. 不需要解引用操作，语法更简洁
4. 原始 `num` **被直接修改**

**比喻：** 就像给别人起个外号，叫外号就是在叫本人。

## 详细对比分析

### 代码执行流程
```cpp
int num = 10;                    // num = 10

// 第一次调用：传值
incrementByValue(num);           // 创建副本，副本从10变成11，原num还是10
cout << num << endl;             // 输出：10

// 第二次调用：传指针  
incrementByPointer(&num);        // 通过指针直接修改num：10 → 11
cout << num << endl;             // 输出：11

// 第三次调用：传引用
incrementByReference(num);       // 通过引用直接修改num：11 → 12
cout << num << endl;             // 输出：12
```

### 内存示意图
```
初始状态：
num [10]  ← 地址：0x1000

调用 incrementByValue(num)：
num [10] (地址0x1000)  →  创建副本 x [10] (地址0x2000) → x变成11
结果：num还是10

调用 incrementByPointer(&num)：
传递地址0x1000 → 指针x指向0x1000 → (*x)++ → num变成11
结果：num变成11

调用 incrementByReference(num)：
x是num的别名 → x++ → num变成12  
结果：num变成12
```

## 三种方式的比较

| 特性      | 传值                 | 传指针                 | 传引用                 |
| ------- | ------------------ | ------------------- | ------------------- |
| 语法      | `func(var)`        | `func(&var)`        | `func(var)`         |
| 参数声明    | `void func(int x)` | `void func(int* x)` | `void func(int& x)` |
| 函数内使用   | 直接使用 `x`           | 解引用 `*x`            | 直接使用 `x`            |
| 是否修改原变量 | ❌ 否                | ✅ 是                 | ✅ 是                 |
| 是否需要取地址 | ❌ 否                | ✅ 是                 | ❌ 否                 |
| 安全性     | 高（隔离）              | 中（需检查空指针）           | 高（不能为空）             |
| 性能      | 低（拷贝开销）            | 高（传地址）              | 高（传地址）              |

## 使用场景建议

### 传值：适合小型数据，且不需要修改原值
```cpp
// 计算平方，不需要修改原值
int square(int x) {
    return x * x;
}
```

### 传指针：需要修改原值，或参数可能为空
```cpp
// 交换两个变量的值
void swap(int* a, int* b) {
    if(a && b) {  // 检查空指针
        int temp = *a;
        *a = *b;
        *b = temp;
    }
}
```

### 传引用：需要修改原值，且希望语法简洁
```cpp
// 修改对象状态
void updateStudent(Student& student) {
    student.grade = 'A';
    student.score += 10;
}
```

## 关键理解点

1. **传值**创建副本，原变量安全但有效率代价
2. **传指针**传递地址，可以修改原值但语法稍复杂
3. **传引用**是"语法糖"，既有指针的效率又有传值的简洁语法
4. 引用必须初始化且不能改变指向，指针可以改变指向

在C++中，**传引用通常是最推荐的方式**，因为它结合了安全性和效率。
